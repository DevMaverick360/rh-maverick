Google Forms → Maverick 360

Guia para quem usa o painel. Pode receber candidaturas sem vaga cadastrada (entrada geral) ou enviar job_code / job_id para associar a uma vaga. O script completo com token atualizado está em Configurações → Integração (área autenticada) — copie dali para o Google Apps Script.

1. URL da API

POST JSON Se abrir esta página a partir do site, o domínio é preenchido automaticamente. 
https://seu-dominio.com/api/cv/submit

2. Autenticação

Cada pedido leva o token de integração no cabeçalho Authorization: Bearer …. O valor correto aparece apenas no painel (não envie o token por e-mail ou redes sociais).

Headers 
Content-Type: application/json
Authorization: Bearer <token em Configurações → Integração>

3. Exemplos de corpo JSON

Entrada geral (sem vaga):

JSON
{
  "form_responses": [
    { "question": "Nome", "answer": "João" },
    { "question": "E-mail", "answer": "joao@exemplo.com" }
  ]
}

Com vaga (use o código definido na vaga no painel):

JSON com job_code
{
  "job_code": "frontend-sp",
  "form_responses": [
    { "question": "E-mail", "answer": "candidato@exemplo.com" }
  ]
}

4. Google Apps Script

Crie o gatilho instalável na função enviarMaverick, origem Do formulário, evento Ao enviar o formulário. Não use onFormSubmit para chamadas HTTP.

O código pronto (com token e URL do seu site) está em Configurações → Integração no Maverick.

Checklist
  • Recolher e-mail no Google Formulário (definições) ou pergunta com e-mail válido.
  • Gatilho apontando exatamente para enviarMaverick.
  • Autorizar o script (rede; Drive se houver upload de CV).
  • Teste com envio real pelo link público do formulário.
